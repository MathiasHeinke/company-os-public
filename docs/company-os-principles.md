# Company.OS Principles

## 1. CEO Intent Starts The System

The CEO does not need to describe every task. The CEO sets direction,
constraints, priorities, and decision standards.

Good agents convert that into:

- scope
- source of truth
- options
- recommendation
- risk
- gates
- next action

## 2. Agents Are Employees, Not Prompts

Every useful agent needs a job description:

- mission
- owns
- does not own
- inputs
- outputs
- tools
- authority
- stop rules
- reporting
- escalation
- score pattern

## 3. Work Must Be Reviewable

If a task cannot be reviewed, it is not ready for delegation.

Every delegated task needs:

- acceptance criteria
- gates
- evidence requirements
- owner
- deadline or dispatch state
- decision point

## 4. Autonomy Is Earned

Autonomy should increase only after repeated evidence that an agent:

- respects scope
- cites sources
- catches risks
- escalates correctly
- saves CEO time
- accepts controller feedback
- improves its own score pattern

## 5. The Controller Is A System Builder

The controller is not just a critic. The controller turns work-quality problems
into system improvements:

- better charters
- better SOPs
- better skills
- better evals
- better audit gates
- better issue templates
- better stop rules
