# 0.7 Entry Plan - Read-Only Command Center And Managed Producer Promotion

Status: draft release entry plan
Date: 2026-05-20
Input baseline: `0.6.5-beta.5`

## Purpose

`0.6.5` proves the runtime substrate: Plane-first dispatch, sandbox workspace
guards, CAO/Controller gates, Raindrop managed-call evidence, prompt-result
evaluation, template seed closure and contract-wizard diagnostics.

`0.7` should not start by building a broad UI. It should start by defining the
read-only data contract that a UI may display, then promoting only the managed
producer surfaces needed to keep that contract auditable.

The target is a read-only Command Center POC that can answer:

- what happened last night
- which workers ran, failed, passed or parked
- which HumanGates need review
- which HG-3 packets Codex/CEO released and which were held for EVE /
  Chief-of-Staff `HG-3.5` review
- which reports, traces and controller decisions support each claim
- what the operator may safely do next

## Decision

The first `0.7` workstream is **read-only evidence composition**, not write
automation and not browser UI polish.

The browser-guided install / AionUI / Hermes layer remains a product surface on
top of Company.OS. It may be evaluated in a sandbox, but it must consume a
Company.OS read model rather than becoming a second ledger, scheduler, memory
store or release authority.

`0.7.1` and `0.7.2` extend this line from internal Command Center readiness to
client self-onboarding readiness. The canonical continuation goal is:

```text
docs/releases/0.7.1-client-self-onboarding-readiness.md
```

## Non-Goals

- no production writes
- no Plane Done transitions
- no direct publish, send, schedule, spend, deploy, schema, RLS, auth or
  service-role changes
- no hosted Raindrop ingestion
- no raw prompts, raw tool payloads, cookies, browser storage or private memory
  in reports
- no autonomous resume from HG-3.5 until EVE / Chief-of-Staff sign evidence is
  explicit
- no UI process with real secrets

## Release Bar For `0.7.0-beta.1`

`0.7.0-beta.1` is eligible only when all of these are true:

1. A deterministic read-only Command Center state packet exists.
2. The packet can be generated from existing Company.OS artifacts:
   Plane-safe inputs, `metrics/agent-events.jsonl`, worker reports, CAO
   verdicts, controller decisions, release evidence and Raindrop summaries.
3. Every visible claim in the packet links to a source artifact.
4. HumanGate review items are surfaced as decision cards or intent-card drafts,
   never as direct execution.
5. At least one scheduled or recurring worker lane has Raindrop-safe producer
   evidence, or is explicitly listed as not-yet-promoted with a blocker.
6. AionUI/Hermes sidecar evaluation consumes only the read-only packet or
   sanitized mock data.
7. Productization readiness passes with zero blockers in guided beta mode.

## Rocket Start Sequence

### RS-18 - Command Center Read Model Contract

Role: `role:cpo` with CTO review.

Outcome: define the canonical read-only state shape for the Command Center.

Implementation seed: `scripts/command-center/command-center-read-model.mjs`
builds a deterministic read-only packet from `metrics/agent-events.jsonl`.
The packet currently covers morning brief totals, worker run cards, HumanGate
queue entries and Raindrop trace summary cards. This is the first executable
slice, not the final UI.

Scope:

- include morning brief state, worker run state, HumanGate queue, report
  drill-down, trace summary cards and intent-card draft shape
- include source-link rules and redaction class per field
- exclude UI implementation and state mutation

Artifacts:

- `docs/releases/0.7a-command-center-read-model.md`
- sanitized sample packet under `reports/examples/command-center-read-model/`
- executable seed: `scripts/command-center/command-center-read-model.mjs`
- UI handoff boundary:
  `docs/integrations/aionui-hermes-command-center-handoff.md`

Gates:

- read-model fields cite existing artifacts only
- no mutable action is represented as direct execution
- CAO review checks source-link and redaction discipline

### RS-19 - Scheduler Producer Promotion For Raindrop

Role: `role:cto`.

Outcome: promote the scheduler / recurring worker launch path from
not-yet-instrumented to managed Raindrop producer, if and only if a real hook
producer exists.

Scope:

- include hook producer metadata and tests
- include runtime preflight coverage
- include one synthetic scheduler-run fixture
- exclude hosted Raindrop ingestion and arbitrary manual sessions

Suggested artifacts:

- update `scripts/orchestration/raindrop-call-adapter.mjs`
- update runtime/scheduler tests that prove hook-shape and builder coverage
- update `docs/operations/raindrop-llm-call-observability.md`

Gates:

- all Raindrop adapter tests pass
- Runtime Dispatcher preflight still blocks missing/broken producer coverage
- reports contain no raw prompts or private payloads

### RS-20 - AionUI/Hermes Sidecar Sandbox Evaluation

Role: `role:coo`.

Outcome: evaluate whether AionUI/Hermes should be adopted, forked, adapted,
parked or rejected for the first read-only operator shell.

Scope:

- include sandbox install/eval only
- include screenshots, capability matrix, security notes, adapter requirements
  and read-only packet consumption feasibility
- exclude real secrets, production credentials, Plane writes, dispatch, Done,
  deploy, publish and private data ingestion

Suggested artifacts:

- `reports/operator-shell/<date>/aionui-hermes-sidecar-eval.md`
- updated recommendation in `docs/strategy/operator-shell-candidate-scorecard.md`

Gates:

- no secrets in screenshots or reports
- every UI claim maps back to the read-only packet or sanitized mock
- final recommendation is `adopt`, `fork`, `adapter-only`, `from-scratch`,
  `park` or `reject`

### RS-21 - Portfolio Scheduler Ranking Model

Role: `role:coo` with CTO review.

Outcome: define ranking-only portfolio scheduling for 100+ contracts without
parallel execution authority.

Scope:

- include scoring inputs, freshness, priority, role, dependency and gate
  blockers
- include why an item is not runnable
- exclude parallel worker execution and automatic dispatch-ready flips

Suggested artifacts:

- `docs/orchestration/portfolio-scheduler-priority-model.md`
- ranking report fixture under `reports/scheduler/portfolio-priority/`

Gates:

- ranking-only output
- no dispatch, no Done, no release authority
- controller can explain every rank from source fields

### RS-22 - Command EVE HG-3.5 Decision Packet Surface

Role: `role:coo` with CEO/Codex and CAO review.

Outcome: define the read-only Command EVE / Chief-of-Staff surface for HG-3
holds and HG-4 founder preparation.

Scope:

- include `CEO_CRITICAL_RELEASE` vs `CEO_CRITICAL_HOLD` display rules
- include simulated HG-3.5 packet shape while EVE is not yet live
- include Founder-facing HG-4 dossier compression rules
- include source-link, challenge, assumption and consequence fields
- exclude release authority, Plane Done, dispatch, deploy, production write
  and autonomous resume

Suggested artifacts:

- `docs/operations/eve-chief-of-staff-hg35-review.md`
- sample HG-3.5 packet under
  `reports/examples/command-center-read-model/hg35-eve-packet.json`
- read-model extension in `scripts/command-center/command-center-read-model.mjs`

Gates:

- every HG-3.5 packet links to CAO/controller evidence and the original
  worker report
- no HG-4 decision is represented as approved without real Founder sign
- simulated EVE review is visibly marked as simulated until the v0.7 runtime
  exists
- Command Center remains read-only

### RS-23 - EVE Founder Intent Operating Layer

Role: `role:coo` with CEO/Codex, CPO, CTO and CAO review.

Outcome: define how EVE turns loose Founder conversation into CEO Delegation
Packets and Plane parent/child worker contracts without becoming the execution
authority.

Scope:

- include Founder Intent Packet and CEO Delegation Packet shape
- include EVE/CEO dialogue protocol and challenge rules
- include AionUI/Hermes action-environment boundary
- include release placement through `1.2.x`, `2.x` and `3.x`
- exclude direct dispatch, release, Done, HG-4 approval, production writes and
  private raw memory ingestion

Suggested artifacts:

- `docs/operations/eve-founder-intent-operating-layer.md`
- Plane parent item for the EVE lane
- child worker contracts for doctrine, packet schema, sidecar UI evaluation,
  adapter/read-model work, CAO authority review and productization/release
  ladder alignment

Gates:

- every child contract uses the canonical parseable worker issue contract
- all child contracts stay `Dispatch: manual` until CEO/Codex promotes them
- AionUI/Hermes consumes only read-only state or sanitized mocks
- HG-4 remains Founder-only

## Recommended Order

Run RS-18 first. Without a read model, UI and producer work has no stable target.

Then run RS-19 and RS-20 in parallel only if their write scopes are isolated.
RS-20 must consume the sanitized sample packet first. RS-21 can follow once the
read model knows which queues and blockers must be visible. RS-22 should start
after RS-18 so EVE/HG-3.5 consumes the same source-linked read model rather than
inventing a second decision ledger. RS-23 can run in planning mode now because
it creates doctrine and manual-dispatch contracts; implementation still waits on
the read-only packet and sidecar sandbox gates.

## Promotion Notes

The current `0.6.5-beta.5` state is sufficient to begin RS-18. It is not yet
enough to claim `0.7.0-beta.1`, because managed producer promotion and the
sidecar absorption decision do not exist yet.

After `0.7.0-beta.1`, the release line should not jump straight into broad
autonomous UI work. The next gated move is `0.7.1` / `0.7.2`: portable install,
client onboarding, stack registry and update routine for a first paid
non-founder pilot.
