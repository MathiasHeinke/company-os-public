# 0.7A Command Center Read Model

Status: executable seed
Date: 2026-05-20
Parent: `docs/releases/0.7-entry-plan.md`

## Purpose

The Command Center read model is the first executable `0.7` artifact. It
reduces Company.OS execution evidence into a read-only packet that a future UI,
AionUI/Hermes sidecar or morning brief can display without owning state.

This is not the UI. It is the contract a UI must consume.

## Executable Source

```bash
node scripts/command-center/command-center-read-model.mjs \
  --events metrics/agent-events.jsonl \
  --format markdown \
  --max-runs 20
```

Write deterministic artifacts:

```bash
node scripts/command-center/command-center-read-model.mjs \
  --events metrics/agent-events.jsonl \
  --out-dir /absolute/path/to/reports/command-center/read-model
```

Build the sanitized UI/mock packet:

```bash
node scripts/command-center/command-center-read-model.mjs \
  --events reports/examples/command-center-read-model/agent-events.example.jsonl \
  --out-dir reports/examples/command-center-read-model \
  --file-stem command-center-read-model.example \
  --generated-at 2026-05-20T12:00:00.000Z \
  --max-runs 10
```

Core module:

```text
scripts/command-center/command-center-read-model-core.mjs
```

## Source Policy

The read model may read only artifact-backed sources:

- `metrics/agent-events.jsonl`
- worker reports referenced by event rows
- CAO verdicts and controller decisions represented in event rows or linked
  artifacts
- Raindrop safe summaries and prompt-result summaries referenced by event rows

It must not read raw prompts, raw model output, browser storage, cookies,
private memory, `.env` files, production systems or Plane directly.

Plane remains the execution ledger. This model is a projection, not a ledger.

Sanitized examples live under:

```text
reports/examples/command-center-read-model/agent-events.example.jsonl
reports/examples/command-center-read-model/command-center-read-model.example.json
reports/examples/command-center-read-model/command-center-read-model.example.md
reports/examples/command-center-read-model/hg35-eve-packet.json
```

AionUI/Hermes and other UI experiments should consume these examples first,
then explicitly opt into live internal ledgers only after a separate contract.

## Sections

### Morning Brief

The morning brief is a reduction layer, not a raw event dump.

It includes:

- number of visible runs
- active runs
- blocked runs
- HumanGate-required runs
- HG-3 CEO/Codex critical releases
- HG-3.5 / EVE packets
- trace-card count
- a one-line headline

### Worker Run Cards

Each run card includes:

- `issue_id`
- `parent_issue_id`
- `run_id`
- `agent`
- `mode`
- `role_owner`
- `department`
- `worker_state`
- `controller_state`
- `issue_state_recommendation`
- `human_gate_state`
- `human_gate_level`
- `decision_gate`
- `merge_state`
- `next_action`
- `blocking_reasons`
- `source_event_ids`
- `artifact_paths`
- `trace_cards`

Every card must carry a source event id. A UI may render the card only if it
can show where the state came from.

### HumanGate Queue

The HumanGate queue is a read-only list of decisions the operator may inspect.
It is not a signing API.

Every entry includes:

- issue id
- run id
- owning role
- next action
- blocking reasons
- source event id
- linked artifacts

Future UI layers may draft intent cards from these entries. They must not
execute the release, sign the gate or resume a worker directly.

### CEO Critical Releases

`ceo_critical_releases` lists `HG-3` release events that Codex/CEO validated
under `CEO_CRITICAL` authority.

Every entry includes:

- issue id
- run id
- release authority
- released by
- founder-prediction confidence
- release validation summary
- source event id
- linked artifacts

This list is not a deploy button. It is evidence that the CEO/Codex gate passed
and that the requested action stayed below HG-4.

### EVE / HG-3.5 Packets

`eve_hg35_packets` lists:

- `HG-3` holds routed to EVE / Chief-of-Staff review
- declared `HG-3.5` pause packets waiting for sign/reject evidence

Until the v0.7 EVE runtime exists, simulated packets must show
`simulated: true`.

Every entry includes:

- originating gate
- target gate
- decision mode
- challenge questions
- assumptions
- consequences
- source event id
- linked artifacts
- blocked actions

This section may prepare or challenge a decision. It must not approve HG-4,
resume a worker or write Plane state.

### Trace Summary Cards

Trace cards link read-model state to Raindrop safe summaries.

The reducer dedupes repeated event pointers so `worker.reported` and
`ledger.run_summarized` do not double-count the same trace artifact.

Prompt-result summaries are represented separately from call summaries.

## Blocked Actions

The packet always declares:

- no Plane writes
- no Plane Done transitions
- no worker dispatch
- no deploy/publish/send/schedule/spend
- no production writes

Any UI or sidecar that consumes this packet inherits those blocked actions.

## Current Smoke

Against the current internal event ledger, the CLI can produce:

```json
{
  "schema": "command-center-read-model/v0",
  "runs": 3,
  "human_gates": 3,
  "trace_cards": 6
}
```

This proves the read-model reducer can see recent worker runs, HumanGate needs
and Raindrop evidence without touching Plane or dispatching workers.

Against the sanitized sample ledger, the committed fixture produces:

```json
{
  "runs": 5,
  "active": 0,
  "blocked": 1,
  "human_gates": 2,
  "hg3_ceo_releases": 1,
  "hg35_packets": 1,
  "trace_cards": 3
}
```

This fixture is the first stable mock input for UI and adapter work.

## AionUI / Hermes Handoff

The UI handoff boundary is documented in:

```text
docs/integrations/aionui-hermes-command-center-handoff.md
```

That document states the source-ownership and installation policy: UI
experiments start as sandbox consumers of the read-only packet. A customized
Company.OS operator shell must be reproducible from GitHub before it becomes a
product surface. The browser/UI process must not hold production secrets,
Plane write credentials or release authority.

## Acceptance Bar

`0.7A` is acceptable when:

- the reducer test suite passes
- the CLI can render JSON and Markdown from live or sanitized event ledgers
- every visible claim links to source event ids or artifact paths
- duplicate trace pointers are deduped
- HG-3 releases surface separately from HG-4 Founder review
- HG-3 holds and HG-3.5 pauses surface as EVE / Chief-of-Staff packets
- Productization Readiness passes in guided beta mode
- no state-changing action is represented as executable

## Next Slice

The next implementation slice should either promote a managed producer surface
into the event ledger or run a sandbox AionUI/Hermes evaluation against the
sanitized packet. It should not build a broad UI before the adapter boundary is
proven.
