# Company.OS 0.6 Growth Beta Entry Plan

Status: released as `0.6.0-beta.1`
Product version: `0.6.0-beta.1`
Next target band: `0.6.x-beta` hardening and first real content pilots

## Purpose

Company.OS 0.6 is the first department-wedge track: Marketing and Publishing OS.
It should not start with publishing automation. It starts with a text-first,
evidence-backed operating lane that can draft, review, queue and explain
marketing work while publication remains explicitly gated.

## Entry Conditions

0.6 work may start when:

- 0.5.1 fresh-history mirror verification is green and the 0.5.2 adaptive
  onboarding safety hardening goal is closed.
- The real GitHub destination action is either completed or explicitly parked
  behind HG-2.5 without blocking internal runtime work.
- Plane remains the canonical execution ledger.
- Marketing/Growth work is represented as parseable Worker Contracts, not
  loose prompts.
- No worker can publish, schedule external posts, send outreach, change spend,
  make regulated claims or mutate production systems without a human gate.

## 0.6 Tracks

| Track | Role | First Artifact | Hard Boundary |
|---|---|---|---|
| Text-first Command Center conversation map | `role:cmo` + `role:cto` | What employees can ask, what sources answers cite, which actions become intent cards | No state mutation from chat |
| Blog workflow | `role:cmo` | Research -> outline -> draft -> claim check -> approval packet | No auto-publish |
| Social approval queue | `role:cmo` | Draft variants, platform notes, approval checklist | No scheduling/posting without HG release |
| Claim/compliance gate | `role:cao` + `role:cmo` | Claim ledger shape and evidence threshold | No medical/legal/regulated claims without explicit gate |
| Analytics feedback loop | `role:cmo` + `role:cfo` | Read-only metric ingest and weekly feedback report | No spend changes |
| Editorial controller review | `role:cao` | Controller rubric for voice, claims, freshness and source quality | CAO does not create content |

## First Worker Slice

Recommended first slice:

```text
Company.OS 0.6 — Marketing/Growth Command Center conversation map
```

Role: `role:cmo`
Mode: `plan`
Agent: Claude or Codex
Workspace: `registry:company-os`
HumanGate: HG-2

Outputs:

- conversation map
- source citation rules
- intent-card boundaries
- sample employee questions
- blocked action catalog
- first three worker contracts for Blog, Social Queue and Claim Gate

Why this first: it creates the employee-facing shape of the 0.6 wedge without
touching any production publishing, analytics or customer systems.

## Suggested Child Contracts

1. `0.6A Marketing/Growth Command Center conversation map`
2. `0.6B Blog workflow approval packet`
3. `0.6C Social approval queue`
4. `0.6D Claim/compliance gate`
5. `0.6E Analytics feedback loop`
6. `0.6F Editorial controller rubric`

## Promotion Criteria

0.6 can become a real beta only after:

- at least three Marketing/Growth worker runs produce reviewable artifacts
  without manual repair
- CAO verifies claim/compliance boundaries
- controller decisions are consistent across repeated drafts
- no external publishing action bypasses a HumanGate
- the Command Center conversation map can explain status, blockers and next
  decisions from Plane/report evidence

## Release Outcome

`0.6.0-beta.1` is the spec-tier Marketing/Growth beta release. It confirms the
0.6A-F workflow, governance and controller artifacts, including Founder HG-4
ratification of the claim/compliance gate as a v0 governance structure.

It does not authorize autonomous publication, external scheduling, outreach
sends, spend changes, production writes or specific regulated claim approvals.
Those remain behind their declared HumanGate boundaries.

## Non-Goals

- no public publishing automation as the first step
- no ad spend or pricing changes
- no customer outreach sends
- no production dashboard deployment
- no voice or realtime surface as canonical control plane
- no replacement for Plane as the execution ledger
