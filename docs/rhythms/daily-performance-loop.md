# Daily Agent Performance Loop

## Purpose

The daily performance loop makes agent work measurable and improvable.

It answers:

- What did each agent do?
- Was the result useful?
- Was the work method strong?
- Did the agent understand CEO intent?
- What should improve tomorrow?
- Should autonomy change?

## Daily Flow

```text
Agent Run
-> Agent Self Review
-> Controller Review
-> Cost Ledger Snapshot
-> CEO Intent Fit Observation
-> Performance Snapshot
-> Improvement Proposal
-> Linear Follow-up
-> Memory / SOP / Skill Patch
-> Next Day Brief
```

## Morning CEO Use

The CEO should not read every raw review.

The morning brief should contain:

- agent performance exceptions
- autonomy recommendations
- repeated weakness patterns
- high-confidence improvement proposals
- human decisions needed
- blocked work that needs CEO judgment
- forgotten work that should be revived, parked or killed
- stale assumptions that can mislead agents
- top ideas from the idea radar that deserve a CEO decision
- AI cost exceptions, unknown-token calls and budget blockers

## Weekly Trend Review

Every week, the controller should identify:

- agents improving
- agents drifting
- recurring gate misses
- recurring CEO-intent mismatches
- SOPs that are too weak
- score patterns that need adjustment
- roles that should be split or merged
