# Company.OS First-Run Checklist

Status: example, copy to `.company-os/first-run-checklist.md`
Use for: turning a technical kit install into a controlled first pilot

## Technical Install

- [ ] Company.OS `VERSION` pinned in `.company-os/install-record.md`.
- [ ] `.company-os/env.example` reviewed; real secrets are outside git.
- [ ] `.company-os/operations/workspace-registry.json` customized.
- [ ] `.company-os/operations/software-stack.md` customized.
- [ ] `.company-os/operations/human-gates.md` customized.
- [ ] Git remote and branch policy confirmed.
- [ ] Plane workspace/project created.
- [ ] Plane `role:*` labels created.

## Company Discovery

- [ ] `.company-os/company-discovery-brief.md` completed.
- [ ] `.company-os/onboarding/eve-boot-packet.json` generated or reviewed.
- [ ] Company name, domain, offer and buyer captured.
- [ ] Founder bottleneck and first department wedge selected.
- [ ] Data sensitivity and blocked lanes reviewed.
- [ ] Business pressure test report created.

## EVE First Run

- [ ] EVE says what it already knows from account/report/intake seed.
- [ ] Founder/operator corrects the seed before new memory is saved.
- [ ] EVE offers one of three next moves: confirm company model, inspect
      existing tools/task sources, or define the first operating goal.
- [ ] EVE inventories existing ledgers, roles, owners and task sources before
      creating new Company.OS structure.
- [ ] Missing tools are classified as now / soon / later, not all required at
      first greeting.
- [ ] Optional Obsidian Brain Interface mode selected:
      `disabled`, `read_cockpit`, `dashboard_bases` or `activity_log_spike`.
- [ ] If Obsidian is enabled, `.company-os/operations/obsidian-brain-interface.md`
      is copied from the example and `.obsidian/` remains local-only by default.

## First Pilot

- [ ] First Plane parent created with source-of-truth links.
- [ ] 3-7 child worker contracts generated with `dispatch: manual`.
- [ ] First worker run is read-only or dry-run.
- [ ] CAO/controller evidence path selected.
- [ ] Morning or weekly reporting destination selected.
- [ ] HumanGate owner approves pilot start.

## Do Not Enable Yet

- [ ] Scheduler autonomy.
- [ ] Production writes.
- [ ] Public publishing or outreach.
- [ ] Spend.
- [ ] Worker durable memory writes.
- [ ] Worker or CAO Plane `Done`.
- [ ] Obsidian community plugins, custom plugins or sensitive Web Viewer usage.
