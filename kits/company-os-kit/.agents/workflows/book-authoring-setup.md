# Book Authoring Setup Workflow

Use when a founder asks EVE to write or help write a serious book.

## Department Intent Packet

```yaml
intent_id: intent.book_authoring_setup
company:
author:
working_title:
topic:
goal:
audience:
desired_frame:
fvbm_status: missing | M0 | M1 | M2 | M3
first_safe_scope: local setup, BookSpec/FVBM gate, discovery research and Frame Brief
human_gate: HG-4 for direction, voice, cover and final publish
```

## EVE Response Shape

1. My read
2. What I need to challenge
3. Missing inputs
4. CEO Delegation Packet
5. Start command
6. Draft parent contract
7. Draft child roster
8. Research-before-outline gate statement
9. First Founder decision card

## Default Child Roster

- CMO parent setup and gate map
- CMO Discovery Research and Frame Grounding
- CMO/CPO Research Gate Remediation when source ledger or proof plan is missing
- CPO outline architecture
- CAO Outline Checkpoint
- CMO chapter draft and sharpening
- CMO claim verification
- CAO blind cross-model audit
- CTO deterministic build and bundle
- CMO compliance polish

All children stay `dispatch: manual` until CEO/Codex approves Stage 0.5 and
Stage 0.65 readiness.

## Stop Rules

- Stop if there is no research dossier, source ledger, proof plan or Frame
  Brief and the next step is official outline. Create only `outline_sketch`
  hypotheses before research.
- Stop before drafting unless the official outline has passed the Outline CAO
  Checkpoint.
- Stop before publish, upload, KDP submit, reader send, schedule or spend.
- Stop before durable memory write.
- Stop before HG-4 voice, cover or publication decisions.
- Stop if FVBM is missing and the founder refuses the M0 seed.
