# Video-First Content Engine Setup Workflow

Use this workflow when a founder, CEO, CMO or operator asks Company.OS to set up
a raw-video content engine.

## Purpose

Turn messy capture intent into a safe drop-folder, CMO parent contract, child
worker contracts, dry-run publisher packages and review gates.

## Sources

- docs/superpowers/specs/2026-05-27-video-first-content-engine-design.md
- docs/orchestration/company-os-video-first-content-engine-department-pack-v0.md
- docs/templates/company-os-department-capability-pack.md
- docs/templates/company-os-capability-pack-eval-rubric.md

## Output

1. My read
2. What I need to challenge
3. Department Intent Packet
4. CEO Delegation Packet
5. Proposed target root and folder structure
6. CMO parent contract
7. Child worker roster
8. Capability boundary
9. Dry-run command plan
10. First HumanGate decision needed

## Stop Rules

- Do not upload, post, schedule, send or spend.
- Do not request publisher credentials.
- Do not read browser storage, private memory or unrelated filesystem paths.
- Do not process HG-3/HG-4 material into publish-ready outputs.
- Do not write production systems.
- Do not mark Done.
