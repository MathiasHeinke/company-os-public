# Command EVE Visual Design

Status: v1 social / marketing visual source
Source site: https://try.command-eve.com
Owner: Founder / CMO / Visual Director

## Purpose

Use this file as the design source for Command EVE social visuals, post images,
carousels, score cards and future Image 2.0 generation prompts.

The public visual system should feel like the website: technical-premium,
bright, calm, operational and sharp. The goal is not decorative AI art. The goal
is to make abstract operating-system ideas immediately legible.

## Core Personality

- modern-minimal
- technical-premium
- founder-led but not influencer-like
- precise, calm, operational
- light interface, deep system
- German B2B credibility with enough orange energy to feel alive

Avoid:

- generic AI robot imagery
- dark cyberpunk
- neon sci-fi
- purple/blue gradient SaaS sludge
- random orbs as decoration
- stock office people
- fake 3D dashboards without a concrete point
- overbranded ad graphics

## Typography

Website source:

- Display: `Instrument Sans`
- Body: `Inter`
- Accent / italic emphasis: `Instrument Serif`

Social visual rule:

- Use large, tight `Instrument Sans` headlines.
- Use `Inter` for labels, notes, source lines and UI chips.
- Use `Instrument Serif Italic` only for one accent word or phrase.
- Letter spacing stays normal except small uppercase labels.
- Never let generated images produce text. Text is deterministic overlay only.

- Mist: `#eef2f8`

- Brand blue light: `#60a5fa`
- Brand blue wash: `#eff6ff`
- Flame orange: `#f97316`
- Flame orange light: `#fb923c`
- Emerald ok: `#10b981`

Usage:

- Blue is the operating-system anchor.
- Orange is the command spark, gate or human decision.
- Green is only for live/ok/pass states.
- Most visuals should be 70-85% paper/white/slate, 10-20% blue, 3-8% orange.

## Shapes And UI Motifs

Use:

- rounded panels, 12-24 px radius depending on format
- 1 px slate/blue borders
- white or very light blue panels
- subtle blue/orange top hairline
- command mark `⌘` in orange
- EVE wordmark in ink/black, not orange
- small status chips: `live`, `Gate`, `Freigabe`, `Queue`, `PASS`
- tabular numbers and queue/progress indicators
- thin connector lines from founder intent to department/worker cards
- clean dashboard fragments, not full fake apps
- EVE voice orb / radial frequency bars as a signature motif

Avoid:

- text-heavy screenshots of the product
- card stacks inside card stacks
- dense charts without a readable takeaway
- decorative bokeh/orb backgrounds
- fake logos of third-party tools unless explicitly approved

## Image Formats

Primary LinkedIn single image:

- 1080 x 1350 portrait
- Safe text area: 96 px margins
- Headline max 9 words
- One supporting line max 18 words
- Optional bottom strip: `Command EVE` + one status chip

LinkedIn carousel:

- 1080 x 1350 per slide
- 5-7 slides max for Command EVE
- Slide 1: sharp POV headline
- Slides 2-5: one mechanism per slide
- Last slide: question or audit prompt

X / generic image:

- 1536 x 1024 or 1200 x 675
- Fewer words, stronger visual mechanism

## Social Visual Templates

### 1. Operating Truth Card

Use for AI governance, costs, human gates and founder control.

Layout:

- top-left: small label, e.g. `AI GOVERNANCE`
- main headline
- three small evidence/decision chips
- right side: simple system diagram or queue fragment
- bottom: `⌘ EVE` + source/date if relevant

### 2. Queue / Work-In-Progress Card

Use for "EVE is doing the work" proof-loop posts.

Layout:

- top bar: `⌘ EVE` black wordmark + green `live`
- central: 3-5 department cards
- one orange card = human gate
- bottom feed: 2-3 event rows

### 3. Founder Burden Map

Use for founder pain, "everything lives in your head", context switching.

Layout:

- central founder/command node
- surrounding chips: Sales, Content, Reporting, Admin, Delivery, Follow-up
- orange warning gate for "Founder approval"
- copy should explain the decision, not advertise the product

### 4. News Interpretation Card

Use for current AI news.

Layout:

- source label: `NEWS SIGNAL`
- headline: what changed
- mechanism: why it matters for small companies
- bottom: "What to inspect next"

Never make it look like a news outlet. It is an operator's interpretation.

### 5. [SOURCE_COMPANY] Boundary Card

Use for [SOURCE_COMPANY] Bio.OS.

Layout:

- calmer, more scientific variant of the same system
- use blue/slate more than orange
- show signal -> context -> human review
- always include: `Not medical advice`
- no treatment, dosing, protocol, vendor or diagnosis imagery

## Prompt Pattern For Generated Backgrounds

Generated image prompt should create only the no-text base scene.
All text is added by deterministic overlay.

Prompt skeleton:

```text
Create a clean technical-premium editorial background for Command EVE, matching
https://try.command-eve.com: bright paper-white interface, slate ink, precise
blue operating-system accents, small orange command/gate accents, modern B2B
software dashboard feel, subtle glass panels, no text, no logos, no humans
unless explicitly requested, no generic robot, no purple gradient, no dark
cyberpunk. Leave a calm safe area for overlay typography.
```

## Review Gates

Before any visual is posted:

- Does it look like `try.command-eve.com`?
- Is there one clear idea in the first second?
- Is orange used as command/gate, not as the whole brand?
- Is the EVE wordmark black/ink when present?
- Is all text deterministic and readable on mobile?
- Is there no fake claim, fake dashboard result or fake integration?
- Does the visual add understanding, not decoration?
